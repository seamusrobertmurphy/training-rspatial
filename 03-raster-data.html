3  Raster Data

3.1 The raster model

A raster represents a surface as a grid of equal cells, each holding one value. Where vector data draw discrete objects, rasters describe things that vary continuously across space, elevation, reflectance, temperature, canopy height, and the things we derive from imagery: land cover, disturbance, biomass. The data model is nothing more than a matrix plus the georeferencing that ties cell (1,1) to a place and fixes the cell size. terra is the modern package for it (Hijmans, 2024).

We build a raster over the study plot by summarising the stems onto a grid. First project the stems and give each a basal area, then lay down a 10-metre grid and total the basal area falling in each cell:

library(terra)
library(sf)

stems_sf <- st_as_sf(read.csv("data/scbi_stems.csv"),
                     coords = c("lon", "lat"), crs = 4326) |>
  st_transform(32617)
stems_sf$ba <- pi * (stems_sf$dbh / 2)^2 / 1e4     # basal area, m² per stem

stems <- vect(stems_sf)
r  <- rast(ext(stems), resolution = 10, crs = "EPSG:32617")
#> Error:
#> ! [rast] empty srs
ba <- rasterize(stems, r, field = "ba", fun = sum, background = 0)
#> Error in `h()`:
#> ! error in evaluating the argument 'y' in selecting a method for function 'rasterize': object 'r' not found
names(ba) <- "basal_area"
#> Error:
#> ! object 'ba' not found
ba
#> Error:
#> ! object 'ba' not found

The printout is the raster data model made concrete: rows and columns, cell size, extent, CRS, and the value range. Each 10-metre cell now holds the total stem basal area within it, a crude but genuine biomass proxy.

3.2 Inspecting rasters

The same handful of accessors describe any raster:

res(ba)                                        # cell size, metres
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'res': object 'ba' not found
ext(ba)                                        # bounding box
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'ext': object 'ba' not found
global(ba, c("min", "max", "mean"))            # value range
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'global': object 'ba' not found
plot(ba, main = "Basal area per 10 m cell (m²)")
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'plot': object 'ba' not found

3.3 Raster algebra

Because a raster is a grid of numbers, arithmetic on rasters is arithmetic on cells. This is map algebra, and it is why continuous surfaces are so easy to work with. Converting basal area to an aboveground-carbon proxy is one expression applied to every cell at once:

carbon <- ba * 12          # illustrative MgC per cell from basal area
#> Error:
#> ! object 'ba' not found
names(carbon) <- "carbon"
#> Error:
#> ! object 'carbon' not found

Logical operators work the same way and return a mask of ones and zeros. A “stocked” mask, cells carrying any measurable basal area, is a single comparison:

stocked <- ba > 0.05
#> Error:
#> ! object 'ba' not found
plot(stocked, main = "Stocked cells")
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'plot': object 'stocked' not found

3.4 Reclassification

Turning a continuous surface into a small number of classes is reclassification, the operation behind every land-cover and zonation map. classify() takes a matrix of from–to–becomes rows and cuts the surface at the thresholds:

rcl <- matrix(c(-Inf, 0.05, 1,     # sparse
                0.05, 0.30, 2,     # moderate
                0.30, Inf, 3),     # dense
              ncol = 3, byrow = TRUE)
zones <- classify(ba, rcl)
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'classify': object 'ba' not found
levels(zones) <- data.frame(id = 1:3, class = c("sparse", "moderate", "dense"))
#> Error:
#> ! object 'zones' not found
plot(zones, main = "Stocking classes")
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'plot': object 'zones' not found

3.5 Extraction

The bridge from raster back to field data is extraction: reading the raster value at each point location. terra::extract() samples the surface wherever a plot, a tree, or a sample sits, which is how gridded covariates enter a plot-level model:

pts <- spatSample(as.polygons(ext(ba), crs = crs(ba)), 5, method = "random")
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'spatSample': error in evaluating the argument 'x' in selecting a method for function 'as.polygons': error in evaluating the argument 'x' in selecting a method for function 'ext': object 'ba' not found
extract(ba, pts)
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'extract': object 'ba' not found

Extraction and reclassification are the two operations that connect the raster and vector worlds, and everything applied in the rest of the book uses one or both.

3.6 Keystone: land-cover reclassification

Remote-sensing verification turns imagery into a land-use / land-cover (LULC) map and then reasons about it: how much forest, how much converted, how the classes change through time. The Bolivia national land-cover work does this at country scale; the mechanics are the reclassification and area accounting shown here, which we later carry into the uncertainty chapter to attach a confidence interval to each class area.

We stand in for a spectral index with a smooth synthetic surface over the plot, classify it into cover types by threshold, then reclassify to the forest / non-forest split that most carbon accounting turns on:

set.seed(1)
xy  <- xyFromCell(r, 1:ncell(r))
#> Error in `h()`:
#> ! error in evaluating the argument 'object' in selecting a method for function 'xyFromCell': object 'r' not found
val <- 0.55 + 0.30 * sin(xy[, 1] / 120) + 0.20 * cos(xy[, 2] / 160) +
       rnorm(ncell(r), 0, 0.05)
#> Error:
#> ! object 'xy' not found
ndvi <- setValues(r, pmin(pmax(val, -0.1, na.rm = TRUE), 0.95))
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'setValues': object 'r' not found
names(ndvi) <- "ndvi"
#> Error:
#> ! object 'ndvi' not found

# thresholds -> cover classes
lulc <- classify(ndvi, matrix(c(-Inf, 0.2, 1,     # water/bare
                                 0.2, 0.5, 2,      # grass/shrub
                                 0.5, Inf, 3),     # forest
                              ncol = 3, byrow = TRUE))
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'classify': object 'ndvi' not found
levels(lulc) <- data.frame(id = 1:3,
                           cover = c("water/bare", "grass/shrub", "forest"))
#> Error:
#> ! object 'lulc' not found
plot(lulc, main = "Land cover from a spectral index")
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'plot': object 'lulc' not found

The class areas are the deliverable. freq() counts cells per class, and multiplying by the cell area gives hectares:

cell_ha <- prod(res(lulc)) / 1e4
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'res': object 'lulc' not found
f <- freq(lulc)
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'freq': object 'lulc' not found
f$area_ha <- round(f$count * cell_ha, 2)
#> Error:
#> ! object 'f' not found
f[, c("value", "count", "area_ha")]
#> Error:
#> ! object 'f' not found

Reclassifying the three classes into a forest mask, the exact step behind a forest-area or deforestation figure, is one more classify():

forest <- classify(lulc, matrix(c(1, 0, 2, 0, 3, 1), ncol = 2, byrow = TRUE))
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'classify': object 'lulc' not found
forest_ha <- global(forest, "sum", na.rm = TRUE)[1, 1] * cell_ha
#> Error in `h()`:
#> ! error in evaluating the argument 'x' in selecting a method for function 'global': object 'forest' not found
round(forest_ha, 1)                             # hectares of forest
#> Error:
#> ! object 'forest_ha' not found

That single number, forest area, is what a project claims and an auditor recomputes. Chapter 7 returns to it to ask how certain the number is, because a class area is only as good as the map’s accuracy.

3.7 Exercises

  1. Build the basal-area raster at 20-metre resolution instead of 10. How do the mean cell value and the number of cells change, and why does total basal area stay the same?
  2. Reclassify the basal-area surface into four stocking classes and report the area in hectares of each. Which class covers the most ground?
  3. Extract the basal-area value at the location of every stem with extract(). Why is the extracted value not the same as the stem’s own basal area?
  4. (Applied.) Change the forest threshold on the synthetic index from 0.5 to 0.45 and recompute forest area. By how many hectares does the estimate move, and what does that sensitivity imply for defining a class boundary in a real LULC map?
